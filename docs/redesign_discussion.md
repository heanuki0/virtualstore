| **B** | 새 2D 일러스트 DALL·E 재생성 (아치, 좌우 존, 중앙 안내 데스크가 한 프레임에) | 중간 (1h) | $0.12 |

## 2. 리빙룸 대형화 + 거실·다이닝 통합 (월드팀 컨셉)

### 현재 상태
- **R01 Conran Livingroom**: 거실 중심 파노라마 (소파/코피 테이블/러그 5개 슬롯)
- 공간감 약함 — 파노라마 내에서 거실 영역만 보임

### 기획자 의도 추정
- *공간의 스케일감*을 전달하고 싶음 → 거실 + 다이닝을 한 뷰에 담기
- "월드팀 컨셉"은 Caliverse 내부 비주얼 레퍼런스 (아마 더 대형 라운지/오픈 플랜 인테리어)
- 첫 랜딩 시 "넓다"라는 인상 우선

### 옵션
| 안 | 내용 | 공수 | 비용 |
|----|------|------|------|
| **A** | 현재 R01 파노라마를 **광각 오픈 플랜 거실+다이닝**으로 재생성 | 중간 (30분) | $0.12 (day) ~ $0.36 (3 variant) |
| **B** | R01 거실 유지, **다이닝 별도 파노라마 R03 추가** | 큼 (1.5h) | $0.36 (3 variant) |
| **C** | 월드팀 컨셉 아트 확보 후 그걸 equirect로 변환 / 또는 Unreal 렌더 | 매우 큼 | 별도 협의 |

### 추천: **A** — 현재 SKU 구성과 핫스팟 재배치로 가능, 공수 저렴

### 결정 필요
- 기존 R01 day/sunset/night 3장 모두 재생성 ($0.36) vs day 1장만 ($0.12)?
- 다이닝 테이블/의자 신규 SKU 추가 필요 (현재 20 SKU에는 다이닝 카테고리 없음)

## 3. 웨이포인트 2~3개 (멀티-씬 네비게이션)

### 현재 상태
- 파노라마 1개 = 1개 시점. 웨이포인트 시스템 없음.

### 기획자 의도 추정
- 기획서 p.6에 "웨이포인트" 명시 (2-3개)
- **첫 랜딩 = 광각 전체 뷰** (공간감 강조)
- **웨이포인트 1 = 소파 근처** (소파/쿠션/조명 근접)
- **웨이포인트 2 = 주방/다이닝 근처** (테이블/식기 근접)

### 구현 방법
| 안 | 내용 | 복잡도 | 비용 |
|----|------|--------|------|
| **A** | Marzipano multi-scene + waypoint hotspot (파노라마 3개 준비) | 중간 | $0.36~1.08 (waypoint×variant) |
| **B** | 파노라마 1장 + "줌 인" CSS transform (유사-웨이포인트) | 작음 | $0 |
| **C** | 3개 파노라마 썸네일 스트립 UI → 클릭 시 교체 | 작음 | $0.36 |

### 추천: **A** — 기획서 의도에 가장 부합, 마제파노 네이티브 기능

### 기술적 검토 포인트
- 각 waypoint마다 hotspot 재정의 필요 (근접 뷰에서는 해당 가구만 보이므로 핫스팟 1-2개로 축소)
- 조명 variant × 3 waypoint = 9장 파노라마 → 비용/시간/용량 증가
- 모바일 성능 고려 (Marzipano는 가볍지만 이미지 3배로 늘면 초기 로드 시간 이슈)
